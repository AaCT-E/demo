# StegDB Monitoring Hooks

This directory contains integration points for the StegVerse StegDB
monitoring system and the broader GCAT/BCAT ecosystem.

## Files

| File | Purpose |
|------|---------|
| `emit_status.py` | CI hook that emits a JSON status report after verification |
| `last_status.json` | Latest emitted status (generated by CI, do not commit) |

## Ecosystem Integration

- **Upstream theory**: [GCAT-BCAT-Engine](https://github.com/GCAT-BCAT-Engine)
- **Paper publishing**: [GCAT-BCAT-Engine/Publisher](https://github.com/GCAT-BCAT-Engine/Publisher)
- **Entity sandbox**: [StegVerse-Labs/StegDB](https://github.com/StegVerse-Labs/StegDB)
- **User-facing SDK**: [StegVerse-Labs/SDK](https://github.com/StegVerse-Labs/SDK)

## Status Schema

The `emit_status.py` script produces a JSON report with:
- `schema_version`: Semantic versioning for the report format
- `repo` / `run_id` / `commit_sha`: GitHub Actions context
- `status`: CI job outcome (success / failure / cancelled)
- `traces`: Array of scenario trace summaries (decision, min separation, recovery)
- `ecosystem`: Links to sibling repos in the GCAT/BCAT network

This format is designed to be ingested by StegDB's entity monitoring
pipeline for health tracking across the ecosystem.
